# 新規アプリ アイデア出し

新しいアプリや記事のアイデアを提案するスキル。
実装はせず、提案と方針決めまでを行う。

## 提案の流れ

1. ユーザーの興味・テーマがあれば聞く（なければこちらから提案）
2. 以下の観点でアイデアを3〜5個提案する
3. ユーザーが選んだら、画面構成・機能を簡単にまとめる
4. 合意が取れたら `new-app` or `new-article` スキルで実装に移る

## 提案時の評価観点

各アイデアに以下の3項目を★で評価して表に入れる：

| 観点 | 説明 |
|------|------|
| **検索需要** | そのキーワードで検索する人がどれくらいいそうか（★★★が多い） |
| **競合の少なさ** | 同様のツールが既にどれくらいあるか（★★★が少ない＝チャンス） |
| **実装しやすさ** | Next.js + クライアントサイドで完結できるか（★★★が簡単） |

## 提案フォーマット

```
### 💡 アイデア：{アプリ名}
{一言説明}

| 検索需要 | 競合の少なさ | 実装しやすさ |
|----------|------------|-------------|
| ★★☆    | ★★★       | ★★★        |

**想定キーワード**: 「○○ 計算」「○○ ツール」
**ターゲット**: こういう人が使う
**収益ポイント**: こういう場面で広告が見られやすい
```

## アイデアの方向性ガイド

### 相性が良いジャンル（Mahiro Appの路線）
- **日常計算系**: BMI、カロリー、年齢計算、日数計算、単位変換
- **ランダム系**: ルーレット、あみだくじ、チーム分け、パスワード生成
- **テキスト系**: 文字数カウント、文章校正、名前メーカー
- **お金系**: 貯金シミュレーター、ローン計算、時給換算
- **生活便利系**: タイマー、カウントダウン、TODO、メモ

### 記事と組み合わせるパターン（SEO強化）
- ツール + 「使い方」記事
- ツール + 「○○とは？」解説記事
- ツール + 「おすすめ○選」まとめ記事

## 既存アプリ（重複を避ける）
- 乱数作成アプリ (`/random`)
- 適当割り勘くん (`/warikan`)
- テニス乱数表アプリ (`/tennis`)

## 注意
- 外部APIに依存しないもの優先（クライアントサイド完結）
- スマホで使いやすいシンプルなUI
- 実装の提案だけで、このスキルではコードは書かない
